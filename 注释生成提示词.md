- 为该 TypeScript 文件中的所有变量、类、方法、属性生成中文注释。请保留原有的英文注释，并在其上方添加中文注释。
- 注释格式要求如下：
  - 为每个变量、类、方法、属性添加对应的中文注释。
  - 对于方法、函数，必须为每个参数添加注释，格式为：@param 参数名 参数类型及说明。
  - 对于返回值也要添加注释：@returns 返回值类型及说明。
  - 中文注释应以自然、简洁、符合中文阅读习惯的方式进行描述。
  - 保留原有的英文注释，并在中文注释下方保留英文注释，避免删除原有内容。

案例1：

- 之前

  ```jsx
  export interface TargetFindOptions {
    /**
     * When true, object detection happens on per-pixel basis rather than on per-bounding-box
     * @type Boolean
     */
    perPixelTargetFind: boolean;

    /**
     * Number of pixels around target pixel to tolerate (consider active) during object detection
     * @type Number
     */
    targetFindTolerance: number;

    /**
     * When true, target detection is skipped. Target detection will return always undefined.
     * click selection won't work anymore, events will fire with no targets.
     * if something is selected before setting it to true, it will be deselected at the first click.
     * area selection will still work. check the `selection` property too.
     * if you deactivate both, you should look into staticCanvas.
     * @type Boolean
     */
    skipTargetFind: boolean;
  }
  ```

- 修改后

  ```jsx
  /**
   * 对象检测相关的选项
   */
  export interface TargetFindOptions {
    /**
     * 指示对象检测是否应基于每个像素而不是每个边界框进行
     *
     * When true, object detection happens on per-pixel basis rather than on per-bounding-box
     * @type Boolean
     */
    perPixelTargetFind: boolean;

    /**
     * 指示在对象检测期间要容忍（视为活动）的目标像素周围的像素数
     *
     * Number of pixels around target pixel to tolerate (consider active) during object detection
     * @type Number
     */
    targetFindTolerance: number;

    /**
     * 如果为 true，则跳过目标检测。目标检测将始终返回未定义。
     * 单击选择将不再起作用，事件将在没有目标的情况下触发。
     * 如果在将其设置为 true 之前选择了某些内容，则第一次单击时将取消选择该内容。
     * 区域选择仍然有效。还要检查“selection”属性。
     * 如果您停用两者，则应该查看 staticCanvas。
     *
     * When true, target detection is skipped. Target detection will return always undefined.
     * click selection won't work anymore, events will fire with no targets.
     * if something is selected before setting it to true, it will be deselected at the first click.
     * area selection will still work. check the `selection` property too.
     * if you deactivate both, you should look into staticCanvas.
     * @type Boolean
     */
    skipTargetFind: boolean;
  }
  ```

案例2：

- 之前

  ```jsx
  export const capValue = (min: number, value: number, max: number) =>
    Math.max(min, Math.min(value, max));
  ```

- 修改后

  ```jsx

  /**
   * 限制数值在指定范围内
   * @param min 最小值
   * @param value 当前值
   * @param max 最大值
   * @returns 限制后的值
   */
  export const capValue = (min: number, value: number, max: number) =>
    Math.max(min, Math.min(value, max));
  ```
